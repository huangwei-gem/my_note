# 🚀 Git一键提交脚本使用说明

## 📋 功能介绍
这个脚本可以帮助你一键完成以下操作：
1. 检查Git仓库状态
2. 添加所有更改的文件到暂存区
3. 提交更改到本地仓库
4. 拉取远程仓库更新（避免冲突）
5. 推送到远程仓库

## 🎯 使用方法

### 基本使用
```cmd
git-push.cmd
```

### 自定义提交信息
```cmd
git-push.cmd "你的提交信息"
```

## 📁 脚本位置
- 脚本文件：`git-push.cmd`
- 远程仓库地址：`https://github.com/huangwei-gem/obsidian_data`

## 🔧 功能特性

### ✅ 自动检测
- 检查当前目录是否为Git仓库
- 自动添加远程仓库（如果不存在）
- 检测文件更改状态

### 📅 智能提交
- 自动生成包含日期时间的提交信息
- 支持自定义提交信息
- 显示详细的操作进度

### 🔄 冲突处理
- 自动拉取远程更新
- 使用rebase策略避免不必要的合并提交
- 冲突时提供清晰的错误提示

### 🛡️ 错误处理
- 网络连接问题检测
- Git凭据验证
- 分支权限检查
- 详细的错误提示和解决方案

## 🚨 注意事项

1. **首次使用**：确保你已经在GitHub上创建了仓库，并且本地Git已配置用户信息
2. **认证问题**：如果推送失败，请检查Git凭据管理器中的GitHub凭据
3. **网络问题**：确保网络连接正常，能够访问GitHub
4. **冲突解决**：如果出现冲突，请手动解决后重新执行脚本

## 🔍 故障排除

### 常见问题

**Q: 推送失败怎么办？**
A: 检查以下几点：
- 网络连接是否正常
- GitHub凭据是否正确配置
- 是否有足够的仓库权限
- 远程仓库是否存在

**Q: 出现冲突怎么办？**
A: 脚本会提示你手动解决冲突，解决后执行：
```cmd
git push origin main
```

**Q: 如何查看提交历史？**
A: 脚本执行完毕后会显示最新的提交信息，你也可以使用：
```cmd
git log --oneline
```

## 📞 技术支持
如遇到问题，请检查：

2. 远程仓库地址是否正确

4. Git凭据配置
